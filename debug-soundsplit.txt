ORIGINAL PROBLEM:
SoundSplit opens and shows the master volume slider. Under "Applications" it
only shows the Electron app itself — it does NOT list Chrome, Firefox, Spotify,
or any other apps that are actively playing audio. The per-app volume control
is the core feature and it's not detecting other audio sessions.

ARCHITECTURE:
- Main process spawns a persistent PowerShell sidecar process
- The sidecar uses Add-Type to compile C# code that calls Windows WASAPI COM APIs
- The C# AudioBridge.Poll() method enumerates all audio sessions, gets their
  PID, process name, volume, peak level, and mute state
- Results are sent back as pipe-delimited lines over stdout
- The bridge polls every 150ms
- Results are broadcast to renderer via IPC_SEND.SOUNDSPLIT_SESSIONS_UPDATED
- Legacy fallback: audio-sessions.ts (execSync per-call, uses -EncodedCommand)

1. ENCODING FIX: Changed sidecar spawn from `-Command` to `-EncodedCommand`

2. SILENT CATCH FIX: Added error logging to C# catch blocks (were previously

3. SLIDER BOUNCE FIX: Added `recentlyChanged` Map<pid, timestamp> to prevent
   poll results from overwriting user-initiated slider changes. Holdoff: 2s.

4. LOGGING: Added `OK:False` warning when sidecar returns failed command.

5. FILE-BASED DEBUG LOGGING: Added `dbg()` function that writes to
   C:\Users\OEM\soundsplit-debug.log with timestamps — logs init, READY,
   sendCommand calls, and stderr output for diagnosis.

6. MULTI-LINE RESULT PARSING: Rewrote processOutput() to handle C# Poll()
   results that arrive as multiple stdout lines (not a single line with
   literal \n). Uses `resultLines` accumulator that collects pipe-delimited
   lines between RESULT: prefix and the next command prefix.

7. C# JOIN FIX: Changed C# `string.Join("\\\\n", lines)` → `string.Join("\\n", lines)`
   and correspondingly changed JS `raw.split('\\n')` → `raw.split('\n')`.
   This fixed the mismatch where the C# side was producing a different
   separator than the JS side was splitting on.

8. STATE FILTER RELAXED: Changed C# from `if (state == 2) continue` (skip
   expired) to `if (state == 2 && peak < 0.001f) continue` — keeps expired
   sessions that still have audio activity.

9. MASTER FORMAT ALIGNED: Changed MASTER line from `MASTER|vol|peak` (3 fields)
   to `MASTER|Master Volume|vol|peak|muted` (5 fields) — matches per-app
   format. JS parser updated to expect 4+ fields for MASTER.

10. COM INTERFACE VTABLE FIX: Rewrote IAudioSessionManager2 interface definition.
    Old version had `NotImpl1(); NotImpl2(); GetSessionEnumerator()` which
    assumed methods 1 and 2 were unused. New version has correct method
    signatures: `GetAudioSessionControl(); GetSimpleAudioVolume();
    GetSessionEnumerator()` matching the actual COM vtable layout.

11. IAudioSessionControl2 FLATTENED: Removed `IAudioSessionControl2 : IAudioSessionControl`
    inheritance pattern (which causes vtable offset issues in C# COM interop).
    Now IAudioSessionControl2 is a standalone interface with all methods listed
    explicitly in correct vtable order, starting with GetState().

12. IAudioEndpointVolume EXPANDED: Added all methods (SetChannelVolumeLevel,
    SetChannelVolumeLevelScalar, GetChannelVolumeLevel, GetChannelVolumeLevelScalar,
    SetMute, GetMute) so vtable offsets are correct for all calls.

13. INVARIANT CULTURE: Added `using System.Globalization` and changed all
    `Math.Round().ToString()` calls to use `CultureInfo.InvariantCulture`
    to prevent locale-dependent decimal separators (e.g., comma vs period)
    from breaking pipe-delimited parsing.

14. FLOAT PARSING FIX: Changed PowerShell `[float]$parts[2]` to
    `[float]::Parse($parts[2], [System.Globalization.CultureInfo]::InvariantCulture)`
    for set_volume and set_master commands — consistent with C# side.

15. PROCESS NAME LOWERCASE: C# now calls `.ToLower()` on process names for
    consistent matching with DISPLAY_NAMES lookup table.

16. VOLUME PREFERENCE PERSISTENCE: Added electron-store based `prefsStore`
    keyed by process name (not PID). Saves volume + mute state on user changes.
    Restores when apps relaunch with new PIDs. Controlled by config flag
    `remember_volumes` (default: true). `restoredApps` Set prevents double-restore.

17. SETMASTER RETURNS BOOL: SetMaster() now returns `bool` instead of `void`
    for consistent OK:True/OK:False response pattern.

18. DESTROY SAFETY: destroy() captures `proc` reference before nulling `this.ps`
    to avoid killing a respawned process in the delayed timeout.

/**
 * SoundSplit Bridge — real per-app audio control via Windows WASAPI.
 *
 * Spawns a persistent PowerShell process that uses C# COM interop to
 * enumerate audio sessions, control volumes, and report VU peaks.
 *
 * Communication: JSON-RPC over stdin/stdout. The PowerShell process
 * runs a loop that:
 *   1. On "poll" command: returns all sessions with volume + peak data
 *   2. On "set_volume pid vol": sets volume for a specific session
 *   3. On "set_mute pid 0|1": sets mute state for a specific session
 *   4. On "set_master vol": sets master volume
 *   5. On "exit": terminates the sidecar
 */

// ─── Service ────────────────────────────────────────────────────────────────

interface SavedVolumePrefs {
  volume: number
  muted: boolean
}

const prefsStore = new Store<Record<string, SavedVolumePrefs>>({
  name: 'soundsplit-prefs',
  clearInvalidConfig: true
})

class SoundSplitBridge {
  private sessions: AudioSession[] = []
  private masterVolume = 1.0
  private masterPeak = 0
  private pollInterval: ReturnType<typeof setInterval> | null = null
  private ps: ChildProcess | null = null
  private ready = false
  private pendingData = ''
  private lastPollResult = ''
  private restoredApps = new Set<string>()
  private pollInFlight = false
  private recentlyChanged = new Map<number, number>()
  private resultLines: string[] | null = null

  init(): void {
    dbg('SoundSplitBridge.init() called')
    if (this.pollInterval) return
    this.spawnSidecar()
  }

  private spawnSidecar(): void {
    if (this.ps) return
    const encoded = Buffer.from(PS_SIDECAR_SCRIPT, 'utf16le').toString('base64')
    this.ps = spawn('powershell.exe', [
      '-NoProfile', '-NonInteractive', '-ExecutionPolicy', 'Bypass',
      '-EncodedCommand', encoded
    ], { stdio: ['pipe', 'pipe', 'pipe'], windowsHide: true })

    this.pendingData = ''
    this.ready = false

    this.ps.stdout?.setEncoding('utf8')
    this.ps.stdout?.on('data', (chunk: string) => {
      this.pendingData += chunk
      this.processOutput()
    })

    this.ps.stderr?.setEncoding('utf8')
    this.ps.stderr?.on('data', (data: string) => {
      dbg(`STDERR: ${data.trim().slice(0, 500)}`)
      if (data.includes('Exception') || data.includes('Error')) {
        console.warn('[SoundSplit] PS stderr:', data.trim().slice(0, 200))
      }
    })

    this.ps.on('exit', (code) => {
      console.log(`[SoundSplit] Sidecar exited with code ${code}`)
      this.ps = null
      this.ready = false
      if (this.pollInterval) {
        console.log('[SoundSplit] Restarting sidecar...')
        setTimeout(() => this.spawnSidecar(), 2000)
      }
    })

    this.ps.on('error', (err) => {
      console.error('[SoundSplit] Failed to spawn sidecar:', err.message)
      this.ps = null
    })

    console.log('[SoundSplit] Spawning PowerShell sidecar...')
  }

  private processOutput(): void {
    const lines = this.pendingData.split('\n')
    this.pendingData = lines.pop() || ''

    for (const line of lines) {
      const trimmed = line.trim()
      if (!trimmed) continue

      const isCommand = trimmed === 'READY' ||
        trimmed.startsWith('RESULT:') ||
        trimmed.startsWith('OK:') ||
        trimmed.startsWith('ERR:')

      if (this.resultLines && isCommand) {
        this.pollInFlight = false
        this.parsePollResult(this.resultLines.join('\n'))
        this.broadcastSessions()
        this.resultLines = null
      }

      if (trimmed === 'READY') {
        this.ready = true
        dbg('Sidecar READY')
        console.log('[SoundSplit] Sidecar ready')
        if (!this.pollInterval) {
          this.pollInterval = setInterval(() => this.poll(), 150)
        }
        continue
      }

      if (trimmed.startsWith('RESULT:')) {
        this.resultLines = [trimmed.slice(7)]
        continue
      }

      if (this.resultLines && trimmed.includes('|')) {
        this.resultLines.push(trimmed)
        continue
      }

      if (this.resultLines) {
        this.pollInFlight = false
        this.parsePollResult(this.resultLines.join('\n'))
        this.broadcastSessions()
        this.resultLines = null
      }

      if (trimmed.startsWith('OK:')) {
        dbg(`Sidecar response: ${trimmed}`)
        continue
      }
    }

    if (this.resultLines && this.pendingData === '') {
      this.pollInFlight = false
      this.parsePollResult(this.resultLines.join('\n'))
      this.broadcastSessions()
      this.resultLines = null
    }
  }

  private shouldRememberVolumes(): boolean {
    try {
      const config = getConfig(ToolId.SoundSplit) as SoundSplitConfig
      return config.remember_volumes
    } catch { return true }
  }

  private savePrefs(processName: string, volume: number, muted: boolean): void {
    if (!this.shouldRememberVolumes()) return
    const key = processName.toLowerCase()
    prefsStore.set(key, { volume, muted })
  }

  private restorePrefs(session: AudioSession): void {
    if (!this.shouldRememberVolumes()) return
    const key = session.name.toLowerCase()
    if (this.restoredApps.has(key)) return
    this.restoredApps.add(key)
    const saved = prefsStore.get(key) as SavedVolumePrefs | undefined
    if (!saved) return
    const volDiff = Math.abs(session.volume - saved.volume) > 0.01
    const muteDiff = session.muted !== saved.muted
    if (volDiff) {
      this.sendCommand(`set_volume ${session.pid} ${saved.volume.toFixed(4)}`)
      session.volume = saved.volume
    }
    if (muteDiff) {
      this.sendCommand(`set_mute ${session.pid} ${saved.muted ? '1' : '0'}`)
      session.muted = saved.muted
    }
  }

  private parsePollResult(raw: string): void {
    const sessions: AudioSession[] = []
    for (const line of raw.split('\n')) {
      const parts = line.split('|')
      if (parts[0] === 'MASTER' && parts.length >= 4) {
        this.masterVolume = parseFloat(parts[2]) || 1
        this.masterPeak = parseFloat(parts[3]) || 0
        continue
      }
      if (parts[0] === 'ERROR') {
        console.warn('[SoundSplit] Poll error:', parts.slice(1).join('|'))
        continue
      }
      if (parts.length < 5) continue
      const pid = parseInt(parts[0])
      const name = parts[1]
      if (isNaN(pid) || pid === 0) continue
      const session: AudioSession = {
        pid, name,
        displayName: formatName(name),
        volume: parseFloat(parts[2]) || 0,
        peak: parseFloat(parts[3]) || 0,
        muted: parts[4] === '1',
        iconPath: null
      }
      this.restorePrefs(session)
      const changedAt = this.recentlyChanged.get(pid)
      if (changedAt) {
        const existing = this.sessions.find((s) => s.pid === pid)
        if (Date.now() - changedAt < 2000 && existing) {
          session.volume = existing.volume
          session.muted = existing.muted
        }
      }
      sessions.push(session)
    }
    const now = Date.now()
    for (const [pid, ts] of this.recentlyChanged) {
      if (now - ts >= 2000) this.recentlyChanged.delete(pid)
    }
    const currentNames = new Set(sessions.map((s) => s.name.toLowerCase()))
    for (const name of this.restoredApps) {
      if (!currentNames.has(name)) this.restoredApps.delete(name)
    }
    this.sessions = sessions
  }

  private poll(): void {
    if (!this.ready || !this.ps || this.ps.killed) return
    if (this.pollInFlight) return
    this.pollInFlight = true
    try { this.ps.stdin?.write('poll\n') }
    catch { this.pollInFlight = false }
  }

  private sendCommand(cmd: string): void {
    if (!this.ready || !this.ps || this.ps.killed) {
      dbg(`sendCommand SKIPPED (ready=${this.ready} ps=${!!this.ps}): ${cmd}`)
      return
    }
    dbg(`sendCommand: ${cmd}`)
    try { this.ps.stdin?.write(cmd + '\n') }
    catch { dbg(`sendCommand FAILED: ${cmd}`) }
  }

  destroy(): void {
    if (this.pollInterval) {
      clearInterval(this.pollInterval)
      this.pollInterval = null
    }
    if (this.ps && !this.ps.killed) {
      const proc = this.ps
      this.ps = null
      try { proc.stdin?.write('exit\n') } catch {}
      setTimeout(() => { if (!proc.killed) proc.kill() }, 1000)
    } else { this.ps = null }
    this.ready = false
  }

  private broadcastSessions(): void {
    const data = this.sessions.map((s) => ({
      pid: s.pid, name: s.name, displayName: s.displayName,
      volume: s.volume, peak: s.peak, muted: s.muted, iconPath: s.iconPath
    }))
    BrowserWindow.getAllWindows().forEach((win) => {
      if (!win.isDestroyed()) {
        win.webContents.send(IPC_SEND.SOUNDSPLIT_SESSIONS_UPDATED, data)
      }
    })
  }

  getSessions(): AudioSession[] {
    return this.sessions.map((s) => ({ ...s }))
  }

  setVolume(pid: number, volume: number): boolean {
    const clamped = Math.max(0, Math.min(1, volume))
    dbg(`setVolume pid=${pid} vol=${clamped.toFixed(3)} ready=${this.ready} ps=${!!this.ps}`)
    this.sendCommand(`set_volume ${pid} ${clamped.toFixed(4)}`)
    const session = this.sessions.find((s) => s.pid === pid)
    if (session) {
      session.volume = clamped
      if (session.muted && clamped > 0) session.muted = false
      this.savePrefs(session.name, session.volume, session.muted)
    }
    this.recentlyChanged.set(pid, Date.now())
    return true
  }

  setMute(pid: number, muted: boolean): boolean {
    this.sendCommand(`set_mute ${pid} ${muted ? '1' : '0'}`)
    const session = this.sessions.find((s) => s.pid === pid)
    if (session) {
      session.muted = muted
      this.savePrefs(session.name, session.volume, session.muted)
    }
    this.recentlyChanged.set(pid, Date.now())
    return true
  }

  getMaster(): MasterAudio {
    return { volume: this.masterVolume, peak: this.masterPeak }
  }

  setMaster(volume: number): boolean {
    const clamped = Math.max(0, Math.min(1, volume))
    this.sendCommand(`set_master ${clamped.toFixed(4)}`)
    this.masterVolume = clamped
    return true
  }
}

OPEN QUESTIONS / NEXT STEPS
================================================================================

1. The C# `string.Join("\\n", lines)` in the PS_SIDECAR_SCRIPT template literal:
   Inside a JS template literal, `\\n` is a literal backslash + n (2 chars).
   When base64-encoded to UTF-16LE and decoded by PowerShell, C# sees `\n`
   which is a real newline in C# string literals. This means Poll() returns
   lines separated by actual newlines, which arrive as separate stdout lines.
   The multi-line RESULT accumulator in processOutput() handles this correctly.

2. $ErrorActionPreference = 'SilentlyContinue' risk: If Add-Type fails, the
   sidecar still writes "READY" and enters the command loop, but [AudioBridge]
   won't exist. The poll would then silently fail. The dbg() stderr logging
   should catch this — check soundsplit-debug.log for Add-Type errors.

3. Legacy fallback (audio-sessions.ts) still has the old IAudioSessionControl2
   inheritance pattern which may have vtable offset issues. Not yet updated
   to match the flattened interface pattern in soundsplit-bridge.ts.

4. COM apartment threading: PowerShell default STA should be fine for WASAPI.
   Not confirmed to be an issue.
